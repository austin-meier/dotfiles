# Writing docs in my style

Apply these rules when producing documentation, READMEs, guides, design notes, PR
descriptions, or other prose.

## Voice & tone

<!-- TODO: describe the target voice. Starter defaults below — replace with real preferences. -->
- Direct and concrete. Lead with what the reader needs; cut throat-clearing intros.
- Plain language over jargon. Define a term once if it's unavoidable.
- Active voice. Second person ("you") for instructions.

## Structure

<!-- TODO: describe preferred document structure. -->
- Start with a one-line summary of what the doc covers.
- Short sections with descriptive headers; scannable over prose-heavy.
- Tables for option/flag/path references; numbered lists for ordered steps.
- Fenced code blocks with a language tag; show the command AND expected output where useful.

## Conventions

<!-- TODO: capture concrete mechanical rules. -->
- Sentence case for headers (not Title Case).
- Wrap prose at a consistent column if the surrounding docs do.
- Link to sources rather than restating them.
